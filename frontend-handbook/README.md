# 📖 About

Most frontend resources teach **frameworks**.

This handbook teaches **engineering**.

Instead of memorizing APIs, you'll understand:

- Why technologies exist.
- The problems they solve.
- The trade-offs behind their design.
- How everything connects together.

Whether you are preparing for interviews or becoming a better engineer, this handbook is designed to help you build a strong foundation that remains valuable even as technologies evolve.

---

# 🎯 Goals

This handbook aims to help you:

- Understand Computer Science fundamentals.
- Master JavaScript deeply.
- Learn TypeScript with confidence.
- Understand how Node.js works internally.
- Learn Browser Internals.
- Master React and Next.js.
- Improve debugging skills.
- Build real projects.
- Prepare for frontend interviews.
- Think like an engineer instead of memorizing tutorials.

# ✨ Philosophy

This project follows five core principles.

## 1. First Principles

Every topic starts with **why**.

Before learning *how React works*, we'll first understand the browser.

Before learning the browser, we'll understand the operating system.

Before the operating system, we'll understand the computer.

Knowledge should build naturally.

---

## 2. Learn by Building Mental Models

Instead of memorizing definitions:

❌ Node.js is a runtime.

We'll build a mental model explaining:

```
JavaScript
      │
      ▼
Runtime
      │
      ▼
Engine
      │
      ▼
Machine Code
      │
      ▼
CPU
```

Understanding lasts longer than memorization.

---

## 3. Learn Through Experiments

Every important topic includes hands-on labs.

Instead of reading:

> `package-lock.json` locks dependency versions.

You'll delete it, regenerate it, and observe the results yourself.

---

## 4. Engineering over Frameworks

Frameworks change.

Engineering principles remain.

This handbook focuses on concepts that stay valuable throughout your career.

---

## 5. Explain Design Decisions

Every technology exists because it solved a problem.

Understanding that problem is often more valuable than learning the syntax.

# 🚀 Getting Started

Clone the repository.

```bash
git clone https://github.com/<your-username>/frontend-engineering-handbook.git
```

Install dependencies.

```bash
npm install
```

Start the documentation website.

```bash
npm start
```

Open your browser.

```
http://localhost:3000
```
