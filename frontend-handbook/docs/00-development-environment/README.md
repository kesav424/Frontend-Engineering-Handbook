# Module 0 — Development Environment & Runtime Fundamentals

> "Before learning React, learn the platform React runs on."

---

## Why This Module Exists

Many developers jump directly into React, Next.js, or TypeScript without understanding the environment those tools run in.

This module builds the foundation you'll use throughout the rest of this handbook.

Instead of memorizing commands, you'll understand **why they exist** and **how they work**.

---

# Learning Objectives

By the end of this module, you should be able to answer:

- What is Node.js?
- What is npm?
- What is a runtime?
- What is the V8 engine?
- Why do we need package.json?
- Why does package-lock.json exist?
- What is node_modules?
- What is the difference between a program and a process?
- Why do browsers and Node.js provide different APIs?

# What You'll Build

Throughout this module you'll:

- Run JavaScript inside Node.js.
- Explore the project structure.
- Understand how npm installs packages.
- Learn why package-lock.json exists.
- Understand how JavaScript reaches the CPU.
- Learn the difference between language, runtime, and engine.

---

# Engineering Principles

This module introduces three principles that you'll see throughout the handbook.

## 1. Learn from First Principles

Don't memorize commands.

Understand why they exist.

---

## 2. Build Mental Models

Every concept should connect to another concept.

Knowledge should form a graph, not isolated facts.

---

## 3. Experiment Before Memorizing

Every important concept should be verified through a lab.

If you haven't experimented with it, you probably don't understand it deeply enough.
